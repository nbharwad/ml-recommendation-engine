# Production Hardening Plan — ML Recommendation Engine

## Context

This plan exists because the recommendation system has reached ~49% implementation completeness and requires a structured path to production. The core serving pipeline (FastAPI orchestrator, circuit breakers, Three-Tier feature store, DLRM/XGBoost/Two-Tower models, Triton serving, re-ranking with MMR) is working and well-constructed. However, a systematic gap analysis revealed 36 specific gaps spanning security blockers, missing streaming infrastructure, incomplete GitOps wiring, and absent operational procedures. This system targets 10M+ products, 1M+ DAU, and 50K QPS at p99 < 75ms — production at that scale requires all of these gaps to be resolved before go-live.

The plan is organized into five sequential phases across eight to ten weeks, each building on the previous. Phases 1-3 are strictly gated: no phase begins until the prior phase's acceptance criteria pass. Phases 4-5 run partially in parallel after Phase 3 completes.

### 1.6 Complete JWT Validation in auth.py

**File:** `services/serving/auth.py` (lines 208-237)

**What to do:**

The current `validate_jwt` function decodes the JWT payload using base64 without verifying the signature (lines 218-237). This means any crafted JWT with a valid structure but a fake role claim would be accepted, granting arbitrary permissions.

Replace the stub implementation with real JWKS-based validation using the `python-jose` library (add to `services/serving/requirements.txt`):

```python
from jose import jwt, JWTError
from jose.exceptions import ExpiredSignatureError
import httpx
import time

_jwks_cache: dict = {}
_jwks_cache_time: float = 0
JWKS_CACHE_TTL_SEC = 3600

async def _get_jwks(jwks_uri: str) -> dict:
    global _jwks_cache, _jwks_cache_time
    if time.time() - _jwks_cache_time < JWKS_CACHE_TTL_SEC and _jwks_cache:
        return _jwks_cache
    async with httpx.AsyncClient() as client:
        resp = await client.get(jwks_uri, timeout=5.0)
        resp.raise_for_status()
        _jwks_cache = resp.json()
        _jwks_cache_time = time.time()
        return _jwks_cache

async def validate_jwt(token: str) -> dict:
    config = RBACConfig()
    jwks = await _get_jwks(config.jwt_jwks_uri)
    try:
        claims = jwt.decode(
            token,
            jwks,
            algorithms=["RS256"],
            audience="recommendation-api",
            issuer=config.jwt_issuer,
        )
    except ExpiredSignatureError:
        raise ValueError("Token expired")
    except JWTError as e:
        raise ValueError(f"Invalid token: {e}")
    return claims
```

The JWKS endpoint should be read from `JWT_JWKS_URI` environment variable (already present in `RBACConfig`). Add a startup check that fetches the JWKS endpoint to fail fast if the auth service is unreachable.

**Why it matters:** Without signature verification, the RBAC system is theater. A malicious user can craft a JWT with `"role": "super_admin"` and gain full administrative access to the API. The role hierarchy and permission maps in `auth.py` are well-designed — they just need a real enforcement layer.

**Acceptance criteria:**
- A token with a valid signature from the configured issuer is accepted
- A token with a tampered payload is rejected with 401
- A token with an expired `exp` claim is rejected with 401
- A token with an unknown issuer is rejected with 401
- Unit test coverage for all four rejection cases
- JWKS is cached with TTL and refreshed on 401 from downstream (key rotation support)

### 4.3 Wire Request Coalescing into main.py

**File:** `services/serving/coalescing.py` (stub exists); `services/serving/main.py`

**What to do:**

`services/serving/coalescing.py` has the `RequestCoalescer` class fully implemented but the `CoalescingMiddleware.__call__` method is empty (lines 350-355). There is also a bug at line 163: `math.log(2)` is referenced but `math` is imported at line 395 (after the class definition). Move the import to the top of the file.

Complete `CoalescingMiddleware.__call__`:
```python
async def __call__(self, scope, receive, send):
    if scope["type"] != "http":
        await self.app(scope, receive, send)
        return
    
    request = Request(scope, receive)
    if request.method == "POST" and request.url.path == "/v1/recommendations":
        body = await request.body()
        req_data = json.loads(body)
        user_id = req_data.get("user_id", "")
        params = {k: v for k, v in req_data.items() if k != "user_id"}
        
        async def execute():
            # Reconstruct request with body already read
            scope["_body"] = body
            response_body = []
            async def send_wrapper(message):
                if message["type"] == "http.response.body":
                    response_body.append(message.get("body", b""))
                await send(message)
            await self.app(scope, receive, send_wrapper)
            return b"".join(response_body)
        
        await self.coalescer.execute("recommendations", user_id, params, execute)
    else:
        await self.app(scope, receive, send)
```

In `main.py`, add the middleware after the CORS middleware:
```python
from coalescing import CoalescingMiddleware, CoalescingConfig

app.add_middleware(
    CoalescingMiddleware,
    config=CoalescingConfig(window_ms=5, max_waiters=100),
)
```

**Why it matters:** At 50K QPS on a home page launch event, many users simultaneously request recommendations. Without coalescing, the same user embedding → retrieval → ranking pipeline runs in parallel for near-identical requests, amplifying load on Milvus and the GPU ranking service. Industry data (Google, Netflix) shows 30-70% backend load reduction.

**Acceptance criteria:**

**Why it matters:** Without automated dependency updates, the team relies on ad-hoc `pip install --upgrade` runs or security alerts. Given the Python ecosystem's pace of CVE publication, a system without automated updates will accumulate vulnerable transitive dependencies within weeks of launch.

**Acceptance criteria:**
- Renovate creates its first dependency update PR within 24 hours of configuration
- Auto-merge works for a patch update (manually trigger by downgrading a minor library version)
- A dependency with a known CVE triggers a vulnerability alert PR immediately (not waiting for the weekly schedule)
